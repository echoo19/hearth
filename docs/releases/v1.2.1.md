# Hearth v1.2.1

Hearth v1.2.1 adds asset-pack intelligence across the CLI and MCP server.
Agents can inspect a downloaded local pack for provenance, authored Tiled
metadata, exact compatibility diagnostics, ordered vision inputs, and an
optional contact sheet before importing it. Tilemaps can use fixed named atlas
frames, and validation now catches unsupported mappings, sources, frames,
transforms, and fake autotile rules.

Embedded agent terminals are resilient across sleep, wake, and brief socket
loss. Hearth detaches the shell, buffers its recent output, and reattaches the
editor automatically instead of discarding the running session.

The built-in art/build guidance now teaches agents to inspect unfamiliar packs,
visually verify their contents, preserve provenance, prefer authored sample
maps, and prove a small scene before production placement. Unsupported
isometric or depth compositions are rejected instead of guessed.

## Downloads

Download the desktop build for your operating system below. macOS builds are
Developer ID signed and notarized by Apple. Windows builds are not code-signed
yet, so SmartScreen may require **More info → Run anyway** on first launch.

The release also includes standalone tools for any Node.js 20 or newer
installation:

- `hearth-cli.mjs` — the full Hearth CLI.
- `hearth-mcp.mjs` — the MCP server for Claude Code and other MCP clients.
- `hearth-player.js` — the web-export player; keep it beside the CLI, or set
  `HEARTH_TOOLS_DIR`, so `hearth export web` can find it.

Documentation: <https://github.com/echoo19/hearth#readme>
